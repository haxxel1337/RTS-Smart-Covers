STEG 5 - Kontrollera integrationens syfte

Integrationens funktion:

Den ska skapa en virtuell smart cover ovanpå en RTS/RFXtrx-cover som bara stödjer:
- open
- stop
- close

Exempel source cover:
cover.officesoversleft2

När användaren lägger till integrationen via:
Settings -> Devices & Services -> Add Integration -> RTS Smart Covers

ska config flow fråga efter:
1. Source cover
   - entity selector med domain cover
   - exempel cover.officesoversleft2

2. Smart cover name
   - textfält
   - exempel Office Covers Left Smart

3. Travel time seconds
   - number selector
   - min 1
   - max 300
   - step 0.5
   - unit s
   - exempel 35

4. Initial assumed position
   - number selector
   - 0-100
   - step 1
   - unit %
   - exempel 100

Den nya covern ska:
- stödja open
- stödja close
- stödja stop
- stödja set_cover_position / slider
- visa current_cover_position 0-100
- använda Home Assistants standard: 0 = closed, 100 = open
- ha assumed_state = true
- använda RestoreEntity för att återställa senaste antagna position efter restart
- uppdatera UI-state medan den rör sig, ca 1 gång per sekund
- skicka open/close/stop till source covern
- automatiskt skicka stop när beräknad tid gått

Logik:
- Om current position är 100 och target är 30:
  - difference = -70
  - duration = 70% av 35 = 24.5s
  - service = cover.close_cover
  - efter 24.5s: cover.stop_cover

- Om current position är 0 och target är 30:
  - difference = 30
  - duration = 30% av 35 = 10.5s
  - service = cover.open_cover
  - efter 10.5s: cover.stop_cover

Viktiga implementation notes:
- All logik ska ligga i Python-koden i integrationen.
- Den får inte kräva input_number, input_select, input_datetime, timer, scripts, automations eller template covers.
- Använd time.monotonic() för tidsberäkning.
- Använd homeassistant.helpers.event.async_call_later för timers.
- Använd CoverEntity och RestoreEntity.
- Använd CoverEntityFeature.OPEN, CLOSE, STOP och SET_POSITION.
- Använd CoverDeviceClass.SHADE som default.
- Om source cover är unavailable eller unknown ska smart covern också vara unavailable.
- Om ett nytt kommando kommer medan covern rör sig ska integrationen först stoppa och räkna fram antagen position, sedan starta ny rörelse.
